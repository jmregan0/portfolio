SmartDocs

SmartDocs is a collaborative note taking application that finds helpful research for you as you write. It accomplishes this by incrementally running your text through a Natural Language Processor and extracting the main entities, relationships, and even sentiments in your text.

The more you write, the smarter it gets.

This application was written to avoid the problem of "context switching" in your workflow. Our team wanted to make a tool that could do the mundane work of gathering data for you, so that the user can stay focused on their writing.

Trend Tracker

This project was built for a hackathon at Fullstack Academy in Chicago. The goal was to produce a small tool using technology that was interesting to us, and to finish it in just a few days.

I wanted to explore D3 data visualizations and populate it with regional Twitter trends by tapping into the Twitter developer API. The result is a bubble chart containing the top current hashtags where the size of each bubble represents each hashtag's respective tweet volume.

This was accomplished by running two servers (one for client code, and one to handle Twitter OAuth). The second server was set up to handle CORS requests from the client code, and then retrieve data for different regions by passing a WOEID (where on earth id) along to the Twitter Trends API. This server would then respond with the data, which the client code would translate into D3.

QuickSource

QuickSource was a project I built in my free time as a lightweight version of a previous project - SmartDocs. While SmartDocs required that you write a portion of text before it would fetch your research, this is a tool that allows you to conduct more manual searches for academic research based on keywords.

The development of this application was driven by the desire to minimize the number of steps between a researcher and the information they need to continue their work.

This is version 1.0 and represents a proof of concept for a fast academic search engine with a minimal interface. Continuing development will include a chrome extension for increased ease of use

  • Technologies: React, Redux, Node, Express, CrossRef API, Mailjet

Yet to Surface...

This project is on the way.

Its development began on the premise that your history is something that you should be able to take ownership of. History is an emergent property of the collective people. It's the record and ledger where we note what has happened, what is important, and what should be remembered. Too often, the perception that our history is being written by "others" causes us to disengage.

This is a space that experiments with the concept of collective authorship by common people.
